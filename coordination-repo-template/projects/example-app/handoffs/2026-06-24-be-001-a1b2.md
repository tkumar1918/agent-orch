## Summary
`GET /orders` used to return a bare JSON array of orders. To support large accounts we
now return a paginated envelope `{ data, page, total }` and accept `?page` and `?limit`.
This is a **breaking** response-shape change for any client that reads the top-level array.

## What you need to do
- [ ] Read orders from `response.data` (was the top-level array).
- [ ] Pass `?page` and `?limit`; render `total` for pagination UI.
- [ ] Until 2026-07-15 you may pin old behavior with `X-API-Version: 1` if you need time.
